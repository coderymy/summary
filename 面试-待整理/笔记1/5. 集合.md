## 1. List、Set和Map的理解

1. linked代表链表
2. hash代表hashTable,无序,速度快

速度

链表->二叉树->哈希表

+ List---有序,可重复

  + ArrayList

    数组的顺序表,连续的空间,长度可变的数组

    查快,增删改慢

  + LinkedList

    双向链表的实现,不连续的空间,每个元素指向上一个和下一个元素

    增删改快,查慢

  + Vector

    与ArrayList一致,只是这个是线程安全的(有锁,故效率要低)

+ Set---无序,不可重复

  + HashSet

    底层结构是哈希表

    hashTable(所以无序,不可重复)

    hashTable增删改查都快

  + LinkedHashSet

    使用hash表存储,同时又在hash表上有链表来维护顺序

    所以是有序的

  + TreeSet

    二叉树的红黑树

    有序,查询速度比list快

    但是没有hash表快

# arrayList和LinkedList

arrayList,是顺序表

linkedList,是双向链表

+ List

  有序,可重复

+ ArrayList

  +  顺序表的实现,连续的空间,长度可变的数组
  + 优点:遍历元素和随机访问元素销量高
  + 添加和删除元素效率低,按照内容查询销量慢

+ LinkedList

  + 双向链表的实现,不连续,上一个指向下一个
  + 遍历和随机访问速度慢
  + 插入、删除元素销量高
